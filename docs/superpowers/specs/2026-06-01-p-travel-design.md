# P의 여행 — 서비스 설계 스펙

> 계획은 없어도 괜찮아. 실시간 관광 빅데이터가 오늘 떠날 소도시를 골라주는 즉흥 여행 서비스

*2025 관광데이터 활용 공모전 · 생성형 AI 활용 관광 프롬프톤 부문*

---

## 1. 서비스 개요

**핵심 플로우:** 출발지 + 4가지 조건 입력 → AI가 소도시 1곳 추천 + 하루 코스 생성

**플랫폼:** 모바일 퍼스트 반응형 웹 (Next.js App Router). 공모전 제출은 Vercel 배포 링크.

**레포:** `mio-visit` (기존 GitHub 레포 사용)

**브랜치 전략:** `main` (배포) / `develop` (개발 통합)

---

## 2. 확정된 디자인 방향

### 비주얼 톤
- **컬러:** 그린/자연 계열 기반. 주요 색상 — `#3D6E4A`(딥 그린), `#7DAE7D`(미디엄 그린), `#EEF4EE`(라이트 그린), `#1A2E1A`(다크)
- **폰트:** Pretendard (가변 폰트, weight 100~900 혼용으로 크기 대비 극대화)
- **사진:** 히어로 영역 풀스크린. 카드 틀에 갇히지 않고 숨쉬게.
- **톤앤매너:** AI 티 안 나게 — 구어체 ("오늘 거기 가면 딱인 이유가 있어요"), 크기 대비 크게, 완벽하게 정렬된 카드 레이아웃 지양

### 화면 흐름
```
/ (온보딩) → /trip (조건 입력) → /result (추천 결과)
```

### 각 화면 상세

**온보딩 (`/`)**
- 로고 "P의 여행" + 태그라인
- "내 위치에서 출발" (GPS) / "직접 입력" 두 버튼
- 최초 1회만 노출. 출발지 저장 후 다음 방문 시 `/trip` 직행

**조건 입력 (`/trip`)**
- 그린 그라데이션 풀스크린 배경
- 바텀 시트가 올라오며 하나씩 질문 (총 4단계)
  1. 언제: 오늘 / 내일 / 이번 주말 / 날짜 직접 선택
  2. 이동수단: 자차 / 대중교통(KTX·버스)
  3. 얼마나: 반나절 / 하루 / 1박 이상
  4. 테마: 감성 / 힐링 / 액티비티 / 맛집 / 문화 (단일 선택)
- 각 단계 상단에 "2 / 4" 진행 표시
- 1~3단계: 선택 즉시 다음 시트로 전환 (별도 "다음" 버튼 없이)
- 4단계(테마): 선택 후 "추천받기" 버튼으로 제출

**로딩 (결과 요청 중)**
- 화면 전환 없이 `/result`로 이동하며 스트림 수신
- 단계별 진행 텍스트가 실시간으로 바뀜:
  - "출발지 기준 반경을 계산하고 있어요"
  - "소도시 후보를 추려내고 있어요"
  - "각 소도시 데이터를 수집하고 있어요"
  - "AI가 오늘 최적 소도시를 고르고 있어요"
  - "코스를 짜고 있어요"

**추천 결과 (`/result`)**
- 히어로: 소도시 풀스크린 사진 + "오늘 딱이에요 ✨" 뱃지 + 소도시명 (크고 굵게)
- AI 설명: 구어체 한 줄, 왼쪽 그린 보더 박스
- 인포 칩: 이동시간 / 날씨 / 혼잡도
- 코스 슬라이드: 오전 → 점심 → 오후, 가로 스크롤
- CTA: "지금 출발할게요" (카카오맵 딥링크) + "다시 ↻" 버튼

## 3. 기능 명세

### F-01. 온보딩 (출발지 설정)

| ID | 기능 | 상세 |
|----|------|------|
| F-01-1 | GPS 자동 감지 | "내 위치에서 출발" 탭 → 브라우저 Geolocation API 요청 → 위도/경도를 `localStorage.departure`에 저장 |
| F-01-2 | 직접 입력 | "직접 입력" 탭 → 텍스트 인풋 + 카카오 주소 검색 → 선택한 주소의 위도/경도 저장 |
| F-01-3 | 온보딩 스킵 | `localStorage.departure` 존재 시 `/` 접근 시 자동으로 `/trip` 리다이렉트 |
| F-01-4 | GPS 거부 처리 | 권한 거부 시 직접 입력 모드로 fallback, 안내 메시지 표시 |

### F-02. 조건 입력 (/trip)

| ID | 기능 | 상세 |
|----|------|------|
| F-02-1 | 바텀 시트 진입 | 페이지 진입 시 1단계 시트 자동 표시. 배경 그린 그라데이션 고정. |
| F-02-2 | 1단계: 언제 | 오늘 / 내일 / 이번 주말 / 날짜 직접 선택. 선택 즉시 2단계로 전환. |
| F-02-3 | 날짜 직접 선택 | 달력 UI. 오늘 이전 날짜 선택 불가. 최대 30일 이내만 선택 가능 (집중률 예측 API 한계). |
| F-02-4 | 2단계: 이동수단 | 자차 / 대중교통. 선택 즉시 3단계로 전환. |
| F-02-5 | 3단계: 얼마나 | 반나절 / 하루 / 1박 이상. 선택 즉시 4단계로 전환. |
| F-02-6 | 4단계: 테마 | 감성 / 힐링 / 액티비티 / 맛집 / 문화 — 단일 선택. 선택 후 "추천받기" 버튼 활성화. |
| F-02-7 | 진행률 표시 | 시트 상단 "2 / 4" 텍스트 + 진행 바. |
| F-02-8 | 뒤로가기 | 시트 내 ← 버튼으로 이전 단계 복귀 가능. 1단계에서 뒤로가기 시 온보딩(`/`)으로 이동. |
| F-02-9 | 상태 보존 | React state로 4단계 선택값 관리. "추천받기" 시 query string으로 `/result`에 전달. |

**query string 형식:**
```
/result?dep=37.5665,126.9780&date=today&transport=car&duration=day&theme=healing
```

### F-03. 추천 엔진 (/api/recommend)

| ID | 기능 | 상세 |
|----|------|------|
| F-03-1 | 반경 계산 | 출발지 + 이동수단 + 시간 → 카카오 모빌리티 API로 실시간 교통 반영 도달 가능 시군구 목록 추출 |
| F-03-2 | 소도시 필터 | 반경 내 시군구에서 8대 대도시 제외 (서울·부산·인천·대구·광주·대전·울산·세종) |
| F-03-3 | 데이터 병렬 수집 | 소도시 후보별 TourAPI 5종 + 기상청 병렬 `Promise.allSettled` 호출. 개별 API 실패 시 해당 데이터 null 처리 후 계속 진행. |
| F-03-4 | Claude 추론 | 수집 데이터 + 사용자 조건을 컨텍스트로 주입. 최적 소도시 1곳 선정 + 추천 이유 1문장 생성. 응답은 JSON 형식 강제. |
| F-03-5 | 코스 생성 | 확정 소도시 내 관광정보 API로 장소 풀 수집 → Claude가 테마·시간에 맞는 오전/점심/오후 코스 구성 |
| F-03-6 | 사진 매칭 | 공모전 수상작 사진 API 우선 → 없으면 관광사진 정보 API → 없으면 "아직 아무도 담지 않은 곳" 텍스트 카드 |
| F-03-7 | 행사 매칭 | 선택 날짜 기준 해당 소도시 행사·축제 API 조회. 행사 있으면 AI 설명과 코스에 반영. |
| F-03-8 | Streaming 응답 | 각 단계별 진행 메시지를 `data: {"type":"status","message":"..."}` 형식으로 stream. 최종 결과는 `data: {"type":"result","data":{...}}` |
| F-03-9 | Mock 모드 | `USE_MOCK=true` 시 전주·담양·무주 중 랜덤 mock 데이터 반환. 스트리밍 딜레이 포함. |

**Claude 응답 JSON 스키마:**
```typescript
{
  city: string,           // "전북 무주"
  reason: string,         // 구어체 추천 이유 1문장
  congestion: "low" | "medium" | "high",
  weather: { condition: string, temp: number },
  travelTime: string,     // "자차 2시간 20분"
  course: [
    { time: "오전" | "점심" | "오후", name: string, description: string, imageUrl?: string }
  ],
  event?: { name: string, description: string, isLastDay: boolean },
  mapUrl: string          // 카카오맵 코스 딥링크
}
```

### 데이터 흐름 (Streaming)
```
POST /api/recommend
  { departure, date, transport, duration, theme }
  ↓
  [Server] 카카오맵 반경 계산 → 소도시 후보 풀
  → stream "반경 계산 중..."
  ↓
  [Server] TourAPI 9종 병렬 호출
  → stream "소도시 데이터 수집 중..."
  ↓
  [Server] Claude API 추론 (후보 + 사용자 조건 컨텍스트)
  → stream "AI가 소도시 고르는 중..."
  ↓
  [Server] 코스 생성 + 사진 매칭
  → 최종 JSON (Recommendation) 반환
  ↓
  [Client] LoadingStatus 실시간 업데이트 → 결과 렌더링
```

### AI 레이어 (`lib/ai.ts`)
```typescript
// provider 교체 시 이 파일만 수정
export async function getRecommendation(context: RecommendContext): Promise<ReadableStream>
```
- 기본: Claude API (`ANTHROPIC_MODEL` 환경변수)
- 교체 가능: OpenAI, Gemini 등

### Mock 모드
- `USE_MOCK=true` 환경변수 시 실제 API 호출 없이 mock 데이터 반환
- API 키 없는 개발 단계에서 전체 흐름 테스트 가능

| 영역 | 기술 |
|------|------|
| 프레임워크 | Next.js 15 (App Router) |
| 스타일링 | Tailwind CSS v4 |
| 폰트 | Pretendard (가변 폰트) |
| AI | Claude API — `lib/ai.ts`로 추상화, provider 교체 가능 |
| 지도/경로 | 카카오맵 API |
| 날씨 | 기상청 단기예보 API |
| 배포 | Vercel (`main` 브랜치 자동 배포) |
