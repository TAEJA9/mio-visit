## 4. 화면 구조

### 4-1. 랜딩 페이지 (`/`)

풀스크린 히어로. 배경 `--cream`. 중앙 정렬.

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│                                                     │
│              P의 여행                               │
│         (font-black, text-6xl, --ink)               │
│                                                     │
│    계획은 없어도 괜찮아.                              │
│    실시간 빅데이터가 오늘의 소도시를 골라줘요.          │
│         (font-normal, --muted)                      │
│                                                     │
│         [ 지금 떠나볼까요  →  ]                      │
│    (bg-sage, text-white, rounded-full, px-8 py-4)   │
│                                                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

- `localStorage.departure` 존재 시 `/trip`으로 자동 리다이렉트
- 버튼 클릭 → `/trip`

---

### 4-2. 조건 입력 + 결과 (`/trip`)

**데스크톱 (768px 이상):** 사이드바 + 오른쪽 패널 2단 레이아웃

```
┌──────────────────┬───────────────────────────────────┐
│  사이드바 (300px) │  오른쪽 패널                       │
│  bg: --sidebar   │  bg: --cream                      │
│                  │                                   │
│  출발지          │  [초기 상태]                        │
│  ── ──────────   │  중앙에 안내 텍스트                  │
│                  │  "조건을 입력하면 소도시를            │
│  날짜            │   추천해드려요"                      │
│  ○ 오늘          │                                   │
│  ○ 내일          │  ─────────────────────            │
│  ○ 이번 주말     │  [로딩 상태]                        │
│                  │  스피너 + 진행 텍스트               │
│  이동수단        │                                   │
│  ○ 자차          │  ─────────────────────            │
│  ○ 대중교통      │  [결과 상태]                        │
│                  │  소도시명 (크게, font-black)         │
│  기간            │  AI 한 줄 이유                      │
│  ○ 반나절        │  인포 칩 (날씨·이동·혼잡도)           │
│  ○ 하루          │  코스 카드 3개 (오전/점심/오후)       │
│  ○ 1박 이상      │  [지금 출발할게요] [다시]            │
│                  │                                   │
│  테마            │                                   │
│  [감성][힐링]    │                                   │
│  [액티비티]      │                                   │
│  [맛집][문화]    │                                   │
│                  │                                   │
│  [추천받기]      │                                   │
└──────────────────┴───────────────────────────────────┘
```

**모바일 (768px 미만):** 사이드바가 상단, 콘텐츠가 하단 (세로 스택)

---

### 4-3. 사이드바 상세

- 배경: `--sidebar` (`#F2EDE4`)
- 패딩: `px-6 py-8`
- 각 조건 섹션: 레이블 + 라디오/칩 선택 UI
- **출발지:** 텍스트 표시 + "변경" 링크 (클릭 시 `/`로)
- **날짜·이동수단·기간:** 라디오 버튼 스타일 (선택 시 `--sage` 색상)
- **테마:** 칩 버튼 (다중 선택 가능하도록 변경 고려, 우선 단일)
- **추천받기 버튼:** `w-full bg-sage text-white rounded-xl py-3`
- 사이드바 너비: `w-72` (288px) 고정, 스크롤 독립

---

### 4-4. 오른쪽 패널 상세

3가지 상태를 하나의 컴포넌트에서 관리:

**초기 상태 (`idle`)**
- 중앙에 조용한 안내: "왼쪽에서 조건을 선택하고 추천받기를 눌러보세요"
- 배경 패턴이나 옅은 일러스트 (선택)

**로딩 상태 (`loading`)**
- 중앙 스피너 (sage 컬러)
- 진행 텍스트 실시간 변경 (기존 5단계 유지)

**결과 상태 (`result`)**
```
소도시명 (text-5xl font-black, --ink)
────────────────────────────────────
AI 추천 이유 (border-l-4 border-sage, --muted)
────────────────────────────────────
인포 칩: [🚗 자차 2시간] [☀️ 맑음 24°] [● 한산]
────────────────────────────────────
오늘의 코스
[오전 카드] [점심 카드] [오후 카드]  ← 가로 스크롤
────────────────────────────────────
[지금 출발할게요 →]  [다시 ↻]
```

## 6. 상태 흐름

```
/  (랜딩)
  └─ "지금 떠나볼까요" 클릭 → localStorage.departure 없으면 출발지 입력 모달
     → /trip 이동

/trip
  ├─ RightPanel: idle (조건 미입력)
  ├─ "추천받기" 클릭
  │   ├─ RightPanel: loading (SSE 스트림 시작)
  │   └─ 결과 도착 → RightPanel: result
  └─ "다시 ↻" → RightPanel: idle
```
